# Lifemode Product Overview

## What Lifemode Is

Lifemode is a mobile-first app for people who want clearer thinking around a life decision, emotional tension, or a stuck situation. The product guides a user through a structured reflection flow, helps them organize what they are feeling, and turns useful takeaways into reusable insight.

This build is currently a UI prototype. It demonstrates the product direction, the main user journeys, and the screen architecture, but it does not yet include backend services or persisted user data.

## Product Goal

The goal of Lifemode is to help users move from uncertainty to clarity.

In practice, that means:

- Helping a user name what is on their mind
- Guiding them through a focused reflection process
- Surfacing patterns across sessions
- Preserving meaningful insights for later reuse

## Primary Experience

The app is built around one central loop:

1. A user opens the app and starts a session.
2. They describe the situation they are facing.
3. Lifemode guides them through reflection and framing.
4. The app helps them reach a decision or next step.
5. The result is saved as a session summary or wisdom entry.

This gives the product a clear before-and-after story: from raw thought to structured clarity.

## Main Product Features

### Guided onboarding

The first-time flow is designed to get users into the experience quickly. It includes:

- A landing page with the core value proposition
- Intent selection to choose the kind of support needed
- Thought capture to state the problem in the user’s own words
- Mode selection for quick clarity or deeper reflection
- Account creation

### Session flow

The session flow is the core of the app. It is designed to feel like a guided conversation rather than a form.

It includes:

- Starting a new session from the home screen
- Freeform chat or reflection input
- Emotional framing and interpretation
- Decision support
- Session summary and completion

### Insights and patterns

Lifemode is not only about solving one problem. It also helps users notice repeated patterns over time.

The insights area shows:

- Repeating themes across sessions
- Pattern summaries
- Comparison views for recurring decisions

### Wisdom archive

Completed sessions can be turned into saved wisdom. This makes the product useful beyond a single conversation by creating a personal archive of what the user has learned.

### Structured paths

For common decision types, Lifemode offers guided paths such as career clarity and relationship decisions. These are designed to reduce friction for users who want a more opinionated starting point.

### Account and plan management

The product also includes the supporting screens a real app needs:

- Profile
- Preferences
- Notifications
- Help and support
- Premium plan upgrade
- Billing management

## Core Value Propositions

### 1. Clarity, not just journaling

Lifemode is not positioned as a blank notebook. It provides structure, prompts, and follow-up logic so the user moves toward action.

### 2. Emotional awareness

The product helps users reflect on what they are feeling, not only what they are deciding. That gives the experience more depth and context.

### 3. Memory over time

The app is designed to recognize patterns and preserve useful lessons, so each session compounds into a broader picture of the user’s situation.

### 4. Decision support

The product is built for moments when a user needs to choose, commit, or let go. Its language and screen flow are oriented around that outcome.

## Product Status

What is already represented in the prototype:

- A complete primary user flow
- Supporting screens for navigation and account management
- Session history and insight surfaces
- Empty states for common dead ends

What is still missing for production:

- Backend APIs
- Real authentication
- Persistent storage
- User sync across devices
- Live AI or coaching integration
- Analytics and event tracking

## Summary

Lifemode is shaping up as a decision-support and reflection product, not just a conversation UI. The current prototype already communicates the core product story: start with a messy thought, guide the user toward clarity, and preserve the result as something they can return to later.
